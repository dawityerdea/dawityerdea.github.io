3D Transmon and Cavity cQED (HFSS / Qiskit Metal)

HFSS eigenmode analysis • field visualization • Q factor extraction • geometry sensitivity (sapphire + JJ placement)

Project overview

This work documents a two-part workflow for modeling a 3D cavity coupled to a transmon-style Josephson element. Project 1 builds the baseline cavity model and validates the eigenmode simulation setup. Project 2 builds directly on Project 1 by modifying the sapphire and Josephson-junction placement, then comparing how mode frequencies, field localization, and Q change with geometry.

What I did (portfolio highlights)
  • Set up HFSS eigenmode simulation (multiple modes, convergence tuning, pass control).
  • Built a baseline cavity + transmon-style geometry using Qiskit Metal → exported to HFSS.
  • Compared eigenmode frequencies between baseline geometry (Project 1) and modified geometry (Project 2).
  • Used E-field plots (top + side views) to interpret where energy concentrates (cavity vs. qubit region).
  • Extracted Q from EigenQ results and interpreted “high-Q storage-like” vs “lossy/readout-like” behavior.
  • Documented practical troubleshooting (convergence sensitivity to max passes) and next steps for nonlinear analysis.
Key results (from Project 2)

After moving the sapphire and Josephson junction location, the two dominant eigenmodes show distinct behavior: Mode 1 appears high-Q (storage/qubit-like), while Mode 2 is much lower-Q (more lossy/readout-like).

Metric Mode 1 Mode 2
Resonant frequency (example pass) ~6.2977 GHz ~7.3358 GHz
Q factor (example pass) ~8.4×107 ~9.84×103
Mode spacing Δf ≈ 1.038 GHz (qualitative indicator for coupling / hybridization changes)
Notes / next steps

Extracting a numeric cross-Kerr (χ) or qubit-cavity coupling (g) generally requires nonlinear modeling (e.g., pyEPR / Josephson nonlinearity). The current portfolio focus is eigenmodes, E-field overlap intuition, and sensitivity of f/Q to geometry changes.

  • Run additional geometry sweeps (sapphire position, JJ location) and plot f and Q trends.
  • Export to a nonlinear workflow (pyEPR) to estimate participation and χ.
  • Connect simulation outcomes to a target readout/storage mode design goal.
